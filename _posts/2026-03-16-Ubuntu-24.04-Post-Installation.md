# Ubuntu 24.04 Post-Installation Notes

## Installation Summary

This document summarizes the initial Ubuntu 24.04 setup, post-installation adjustments, and troubleshooting steps performed after installing Ubuntu alongside Windows.

---

Ubuntu 24.04 was installed successfully in a dual-boot configuration alongside Windows.

### Key installation decisions
- Installation mode: **Install Ubuntu alongside Windows Boot Manager**
- Installer mode: **Interactive**
- Package selection: **Default**
- Third-party software: **Enabled**
- Boot mode: **UEFI**
- Installation media: **Bootable USB created with Rufus**

---

## Bootable USB Creation

A bootable Ubuntu USB was created from Windows using **Rufus**.

### Recommended Rufus settings
- **Device:** selected USB drive
- **Boot selection:** Ubuntu ISO
- **Partition scheme:** GPT
- **Target system:** UEFI
- **File system:** FAT32
- **Write mode:** ISO Image Mode (recommended)

---

- **Ubuntu (safe graphics)**

This allowed the installation process to continue successfully.

### Important note
Safe Graphics mode is only a temporary installation/boot workaround. It does **not** mean Ubuntu will permanently use poor graphics quality.

# Installing Ubuntu — A Clear, Natural Step-by-Step Guide

## Introduction

This document explains how to install Ubuntu in a way that feels natural and easy to follow, without turning the process into a rigid checklist. The goal is to give you a practical, human-readable guide that makes sense whether you are installing Ubuntu for the first time, moving from Windows, or simply documenting the process for future use.

Ubuntu installation is not especially difficult, but it does require attention in a few important areas, especially when you are dealing with disk layout, boot mode, and the choice between dual boot and a full Linux setup. Once those parts are understood, the rest of the process becomes much more straightforward.

---

## Before You Begin

Before starting the installation, it helps to decide what kind of setup you actually want. In most cases, people fall into one of two categories.

The first option is a **dual boot** installation. In that setup, Ubuntu and Windows both remain on the machine, and you choose which one to start each time the computer boots. This is often the safer option for someone who is still transitioning, because it allows you to keep Windows available while learning Linux.

The second option is a **full Ubuntu installation**, where Ubuntu becomes the only operating system on the computer. This is usually the cleaner and simpler long-term setup, but it also means Windows and everything on that disk can be removed if you do not prepare carefully.

No matter which route you choose, the most important first step is backing up anything you care about. That includes documents, photos, browser data, SSH keys, work projects, notes, and anything else you would regret losing. If disk partitions are going to be changed, backup is not optional.

---

## What You Need

To install Ubuntu comfortably, you should have a USB flash drive with at least 8 GB of space, a reliable internet connection if possible, the Ubuntu ISO image, and a tool for creating a bootable USB drive. Just as important as the technical requirements is having a little clarity before you start: know whether you want dual boot or a full installation, and know which disk you are working with.

For most people, an Ubuntu **LTS release** is the best choice. LTS stands for Long-Term Support, and those releases are generally more stable and better suited for everyday work, study, and system administration practice.

---

## Downloading the Ubuntu Image

The installation process begins with downloading the Ubuntu ISO file from the official Ubuntu website. Once the file is downloaded, it is good practice to verify that it was downloaded correctly. On Linux, this can be done with a checksum command such as:

```bash
sha256sum ubuntu-xx.xx-desktop-amd64.iso
```

The resulting hash should match the checksum published on the Ubuntu website. If it does, you know the file is intact and safe to use.

This step is often skipped by beginners, but it is worth including in proper documentation because corrupted installation media can cause confusing and unnecessary problems later.

---

## Creating a Bootable USB Drive

Once the ISO image is ready, the next step is to write it to a USB drive so the computer can boot from it. If you are coming from Windows, tools like **Rufus** or **balenaEtcher** are common choices. If you are already on Linux, you can use **balenaEtcher**, **Startup Disk Creator**, or the `dd` command.

If you use `dd`, be careful. It is powerful, but it does not forgive mistakes. First identify the USB device with:

```bash
lsblk
```

Then write the ISO image to the correct device:

```bash
sudo dd if=ubuntu-xx.xx-desktop-amd64.iso of=/dev/sdX bs=4M status=progress oflag=sync
```

In that command, `sdX` must be replaced with the actual USB device. Choosing the wrong disk can overwrite real data, so this is one of the few steps where patience matters more than speed.

After the writing process finishes, safely remove the USB drive.

---

## Booting from the USB Drive

With the USB prepared, restart the computer and enter the BIOS or UEFI firmware settings, or use the one-time boot menu if your machine provides one. The exact key depends on the manufacturer, but common keys include `F2`, `F10`, `F12`, `ESC`, or `DEL`.

At this stage, what matters most is that the machine is allowed to boot from USB and that the boot mode is correct. On modern systems, that usually means **UEFI**. If Windows is already installed in UEFI mode, Ubuntu should also be installed in UEFI mode. Mixing UEFI and legacy BIOS modes on the same machine is one of the most common reasons dual boot setups become messy.

In some cases, **Secure Boot** may also affect the process. Often Ubuntu works fine with it, but if hardware or driver issues appear during installation, temporarily disabling Secure Boot can help with troubleshooting.

---

## Trying Ubuntu Before Installing

When the system boots from the USB drive, Ubuntu usually offers two choices: trying Ubuntu without installing it, or starting the installation immediately.

The live session is useful because it lets you test the machine first. You can check whether Wi-Fi works, whether the display resolution looks correct, whether the touchpad behaves properly, and whether the sound and graphics are functioning normally. This small test can save a lot of frustration, especially on laptops or unfamiliar hardware.

If everything looks good, you can launch the installer from the desktop environment.

---

## Starting the Installation

Once the installer opens, Ubuntu guides you through the setup in a fairly logical order. You begin by selecting the installation language. Although many people choose their native language, using English can make life easier later because most Linux documentation, troubleshooting resources, and community support are written in English.

Next comes the keyboard layout. This is worth testing carefully in the provided text field rather than simply assuming it is correct. A wrong keyboard layout is a small mistake that becomes very annoying later, especially when passwords contain symbols that are mapped differently.

If the installer offers an internet connection at this point, connecting is usually a good idea. Installing updates and third-party drivers during setup can save time after the first boot.

---

## Choosing the Installation Style

Ubuntu typically offers either a **normal installation** or a **minimal installation**. A normal installation includes a wider set of default applications, while a minimal installation provides a leaner system with fewer preinstalled tools.

For someone who wants a cleaner environment and plans to build the system intentionally, minimal installation is often the better choice. For someone who wants a more ready-to-use desktop immediately after setup, normal installation is more convenient.

You may also see options to install updates during the installation and to install third-party software for graphics, Wi-Fi, and related hardware. In most real-world scenarios, enabling both is sensible, especially on laptops or systems using hardware that depends on proprietary drivers.

---

## Understanding the Disk Options

The disk section is the most important part of the installation, because it determines what happens to your existing data and how Ubuntu will coexist with anything already on the machine.

If you choose to erase the disk and install Ubuntu, the installer will use the selected disk for Ubuntu only. This is appropriate when you are doing a clean Linux installation and no longer need the previous operating system.

If Ubuntu detects Windows and offers to install alongside it, that creates a dual boot configuration. In that case, the installer resizes partitions and prepares both systems to live on the same machine. For many users, this is the most comfortable way to move into Linux because it avoids a full commitment on day one.

There is also a manual partitioning option, usually called **Something else**. That is the right choice when you want full control over the disk layout, for example if you want a separate `/home` partition or you simply want to understand exactly how the system is being placed on disk.

---

## Manual Partitioning in a Sensible Way

If you choose to partition manually on a UEFI-based system, there are a few core ideas to understand.

The system usually needs access to an existing **EFI System Partition**, which is typically formatted as FAT32. That partition contains boot-related files. In many cases, especially in a dual boot setup, it already exists and does not need to be recreated.

Ubuntu itself needs a root partition mounted at `/`, usually formatted as `ext4`. Giving this partition at least 25–30 GB is a practical minimum for a comfortable desktop installation.

Some users also create a separate `/home` partition so that personal files are isolated from the main system files. This is optional but useful in some long-term setups.

Swap is handled differently than it used to be. Modern Ubuntu releases often create a swap file automatically, which means a dedicated swap partition is no longer required in many cases.

The main thing is not to overcomplicate the layout. For most users, a simple and clean partition scheme is better than an overly clever one.

---

## Bootloader and Boot Mode

During installation, Ubuntu installs its bootloader so the system can start properly. On UEFI systems, this is handled through the EFI partition. What matters here is consistency: if the machine and existing systems are using UEFI, then Ubuntu should be installed in UEFI mode as well.

If that consistency is broken, the result is often a boot menu that does not behave correctly, missing entries for Windows, or a machine that boots only one system even though both are installed.

This is why understanding the machine’s boot mode before installation is more important than memorizing steps.

---

## Final Confirmation Before Changes Are Written

Before Ubuntu begins writing changes to disk, it shows a summary of what is about to happen. This is the last safe moment to confirm that the correct disk has been selected, the correct partitions are being formatted, and the setup matches your intention.

This is not a step to rush through. Good documentation should say this clearly: once those disk changes are confirmed, the installation becomes real.

---

## Time Zone and User Creation

After the storage decisions are made, the rest of the installer is simple. You choose the correct time zone, which affects system time, logs, and scheduled tasks, and then create your user account.

At that point you enter your full name, machine name, username, and password. A simple, clean username is usually best. Strong passwords are worth using from the start, especially if this system will later be used for administration, SSH access, scripting work, or lab environments.

Ubuntu also asks whether you want automatic login or whether the system should require a password each time. For general desktop convenience, automatic login may be tempting, but for a more disciplined and secure setup, requiring the password is the better habit.

---

## Waiting for the Installation to Finish

Once everything is confirmed, Ubuntu copies files, installs the base system, configures the bootloader, and prepares the machine for its first real boot. The process may take a little time depending on the hardware, but conceptually nothing complicated is happening at this point. The important decisions were already made earlier.

When the installer finishes, it asks for a restart. At that moment, remove the USB drive when prompted so the machine does not boot back into the installer again.

---

## First Boot into Ubuntu

After restart, the machine will either boot directly into Ubuntu or show a GRUB menu if dual boot is configured. In a dual boot environment, GRUB allows you to choose between Ubuntu and Windows.

Once Ubuntu starts, log in with the user account you created. On the first login, you may see a few welcome or setup screens related to privacy preferences, online accounts, appearance, or optional services. These are minor compared to the installation itself and can usually be handled calmly without much risk.

---

## The First Things to Do After Installation

Once you are inside the system, the first practical step is to update it. A fresh installation should be brought fully up to date before you begin customizing or building on top of it.

```bash
sudo apt update
sudo apt upgrade -y
```

In some cases, you may also want:

```bash
sudo apt full-upgrade -y
```

And then reboot the system:

```bash
sudo reboot
```

After that, it is helpful to verify a few basics. You can check the Ubuntu release, the kernel version, the visible disks and partitions, and whether the system was booted in UEFI mode.

```bash
lsb_release -a
uname -r
lsblk
test -d /sys/firmware/efi && echo UEFI || echo BIOS
```

These commands are simple, but they give immediate confirmation that the installation is healthy and aligned with what you expected.

---

## Common Post-Installation Adjustments

After the system is working, there are usually a few practical adjustments worth making. Language settings, keyboard layout, display scaling, additional drivers, browser setup, SSH, terminal tools, and basic utilities often come first.

A small starter package set might look like this:

```bash
sudo apt install -y curl wget git vim htop net-tools tree unzip openssh-server tmux btop
```

That gives you a very solid base for learning, administration, and general Linux work.

If Wi-Fi, graphics, or audio are not working correctly, the first place to check is Ubuntu’s additional drivers section. Hardware issues after installation are often not installation failures at all, but simply missing or inactive drivers.

Useful diagnostic commands include:

```bash
lspci
lsusb
ip a
```

---

## If You Are Using Dual Boot

Dual boot works well when it is set up cleanly, but a few problems appear more often than others. If GRUB does not show up, if Windows boots directly without offering Ubuntu, or if one operating system seems to disappear, the cause is often related to boot mode mismatch, boot order priority in UEFI, or a bootloader issue.

That is why it is worth confirming early that both systems are using the same mode. On modern hardware, that almost always means UEFI.

Dual boot is a practical transitional setup, but it does add some complexity. That complexity is normal, and it is one of the reasons some users eventually move to a full Ubuntu installation later.

---

## If You Later Decide to Move to Full Ubuntu

Many people begin with dual boot and then decide, after gaining confidence, to move fully into Linux. That transition is possible, but it should still be approached carefully. The cleanest method is often to back up important data and perform a fresh Ubuntu-only installation rather than trying to reshape an old dual boot system into something new.

That approach avoids leftover boot complications and gives you a simpler machine to maintain.

---

## Why Ubuntu Is a Good Base for System Administration

Once Ubuntu is installed and working properly, it becomes more than just a desktop operating system. It becomes a practical Linux environment where you can learn shell usage, package management, services, logging, SSH, networking, virtualization, scripting, and automation.

That makes it a strong platform not only for everyday use, but also for building the kind of skills that matter in system administration. It is stable enough for real work and accessible enough for learning.

---

## Closing Thoughts

Installing Ubuntu becomes much less intimidating when the process is understood as a sequence of sensible decisions rather than a list of commands to memorize. The real points of attention are the installation type, the disk layout, and the boot mode. Once those are handled correctly, the rest of the setup is usually smooth.

A good installation guide should feel readable, calm, and practical. It should not force the reader through robotic numbered steps when a natural explanation is clearer. That is the spirit of this document: to explain the process in a way that a person can actually follow and understand.

If you plan to continue toward Linux administration, home lab work, virtualization, or infrastructure practice, then installing Ubuntu is not the end of the process. It is the foundation.
